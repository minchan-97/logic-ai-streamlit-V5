# Logic AI v5

v4에 **마르코프 도메인 가드레일** + **위치별 추적** 옵션을 추가한 버전.

## v4 → v5 변경 사항

**새 모듈 1**: `markov_guardrail.py` — 도메인 자연스러움 점수기
- 사용자가 자기 도메인 코퍼스(.txt)를 업로드
- 마르코프 n-gram 모델 학습 (보통 n=2)
- LLM 응답에 log P(token | context) 평균 점수 매김
- 점수가 임계값보다 낮으면 "도메인 밖 표현" 경고

**새 모듈 2**: `state_path.py` — 위치별 추적 (실험적)
- 마르코프 context를 임베딩 좌표로 매핑
- 학습된 천이 그래프 위에 응답 경로 그리기
- 토큰별로 "그래프 안인가 밖인가" 진단
- 첫 점프 위치 + 점프 단어들 표시
- 어디서 환각이 시작됐는지 토큰 단위 추적

**v4 기본 흐름 유지**:
- 두 system prompt 일관성 비교
- OpenAI 임베딩 mismatch 계산
- Safety 누적 차단
- Cloud N회 호출 옵션

**모두 사이드바 토글로 격리, 기본 OFF.**

## 발상의 출처

오늘 실험으로 발견:
1. 마르코프는 n 커지면 결정성 89%로 lookup table화 (실험 2b)
2. 학습 데이터 밖 시드는 거의 받지 못함 (coverage n=3에서 0.05%)
3. **이 "한계"를 "도메인 신호기"로 뒤집을 수 있다** (실험 4, 5)

마르코프 가드레일은 환각 탐지가 아니라 **도메인 표현 자연스러움 점수기**.
v4의 의미 일관성과는 **다른 종류의 신호**.

## 정직한 한계

- **환각 탐지기가 아님**: "강아지 다리 6개"처럼 표면이 자연스러운
  사실 오류는 못 잡음
- 코퍼스가 작으면 자연스러운 응답도 거부 (가짜 negative 많음)
- LLM 일반 어휘가 도메인 코퍼스 어휘와 다르면 거의 다 -10점
- 표면 자연스러움 ≠ 사실 정확성

## 사용 예

도메인 적용 시나리오:
- 법률 도구: 표준약관 코퍼스 → "이 응답은 법률 표현인가"
- 의료 도구: 의료 매뉴얼 → "이 응답은 의료 표현인가"
- 교사 도구: 교육 매뉴얼 → "이 응답은 교육 행정 표현인가"

가드레일은 **v4 mismatch와 함께** 봐야 가치 있음.
- mismatch 높음 + 가드레일 통과 → LLM 자체 헷갈림 (v4가 잡음)
- mismatch 낮음 + 가드레일 실패 → 도메인 밖 표현 (가드레일이 잡음)
- 둘 다 잡힘 → 강한 신호
- 둘 다 통과 → 그래도 환각일 수 있음 (표면 자연 사실 오류)
